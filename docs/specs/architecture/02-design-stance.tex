\section{Positions}

The shape of the problem matters more than the domain. A few positions this repo takes:

\begin{itemize}[leftmargin=1.2em]
  \item \textbf{The LLM as a kernel, not an application.} The orchestrator is the OS; cortices are processes; tools are syscalls; prompts are binaries on disk. The recursive research loop in Thalamus is closer to the autoresearch pattern that inspired this repo; the kernel analogy is the engineering metaphor used to explain how the pieces are wired.
  \item \textbf{Swarms of small models beat one big one for retrieval.} Up to 50 nano workers crawl and summarize in parallel; a stronger curator dedupes and ranks. Orders of magnitude cheaper than a single strong model per source, bounded latency, easier to cap.
  \item \textbf{Bounded agents, not free-form ones.} Every cortex declares its skills, tools, cost budget and depth cap. Guardrails live in the orchestrator, in code, not in prompts.
  \item \textbf{Deterministic layer beneath the LLMs.} Drizzle ORM, typed repositories, transactional resolutions, structured findings. The model drafts; the system commits. No ad-hoc SQL from the agent.
  \item \textbf{Human-in-the-loop as a first-class citizen.} Sweep is designed around a reviewer, not around autonomy. Accept/reject signals on findings feed back into the next swarm run's prompt per category. Poor-man's RLHF --- calibration without a training run.
  \item \textbf{Observability from day one.} Structured logs, Prometheus counters, per-step traces. Cost and latency instrumented per cortex, per source, per skill.
  \item \textbf{Skills as files, not strings.} Each skill is a markdown prompt versioned with the code. Diffable in PRs, reviewable by non-engineers.
  \item \textbf{Testability end-to-end.} 5-layer architecture, Drizzle-typed repos, isolated services, vitest workspace with unit / integration / e2e.
\end{itemize}
